    Docker

    Docker - Containerized Apache Web Server

    A Docker containerized Apache web server is an instance of the Apache HTTP server running inside a Docker container. This setup encapsulates the web server along with all its dependencies, configurations, and libraries, providing an isolated and consistent environment across different systems. By using Docker, developers can easily deploy, scale, and manage the Apache web server, ensuring a streamlined and efficient workflow. This approach also simplifies version control and improves the portability and reproducibility of the web server environment.

    AWS

    Static Website hosted on S3 Bucket

    Hosting a static website on an Amazon S3 bucket involves storing the site's HTML, CSS, JavaScript, and other static files in an S3 bucket and configuring the bucket for web hosting. Amazon S3 provides a highly available and scalable storage solution, which allows these files to be served directly to users via HTTP/HTTPS. This setup is cost-effective, easy to manage, and can leverage AWS's global content delivery network (CDN) through Amazon CloudFront to improve performance and reliability for users around the world.

    AWS

    Serverless Webapp

    A serverless web app is an application that relies on cloud services to manage the server infrastructure, allowing developers to focus solely on writing code. In a serverless architecture, backend services such as databases, authentication, and server-side logic are provided by cloud providers (e.g., AWS Lambda, Google Cloud Functions, Azure Functions) on a pay-as-you-go basis. This approach eliminates the need for server management, automatically scales resources based on demand, and can significantly reduce operational costs. Serverless web apps are ideal for dynamic, scalable applications where the infrastructure can seamlessly handle varying workloads.
